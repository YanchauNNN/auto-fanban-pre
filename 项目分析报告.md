# CNPE 图纸处理系统 - 项目分析报告

## 一、项目概述

### 1.1 项目简介
这是一个**内网 Web + Python 后端的 DWG 批处理系统**，用于中国核电工程有限公司（CNPE）的图纸自动化处理。

### 1.2 核心功能
系统支持两条主要流水线：

**流水线 A：交付包生成（核心）**
- 输入：DWG 文件 + 项目号 + 文档生成参数
- 处理：DWG→DXF 转换 → 图框检测 → 图签字段提取 → A4多页成组 → 拆分/裁切 → 重命名
- 输出：
  - package.zip（包含：拆分后的 PDF+DWG、封面、目录、设计文件、manifest.json）
  - IED计划.xlsx（单独输出，不打包）

**流水线 B：词库检查/替换**
- 输入：DWG 文件 + 项目号 + 词库 Excel
- 输出：检查报告 + 可选替换后的结果

### 1.3 技术栈
- **后端语言**：Python 3.13+
- **Web 框架**：FastAPI
- **CAD 处理**：ezdxf（DXF 解析）、ODA File Converter（DWG↔DXF 转换）
- **Office 文档**：python-docx（Word）、openpyxl（Excel）
- **PDF 处理**：pypdf、Office COM 自动化
- **配置管理**：Pydantic、PyYAML

## 四、项目架构分析

### 4.1 目录结构
```
auto-fanban-pre/
├── backend/                    # Python 后端
│   ├── src/
│   │   ├── interfaces.py       # 接口定义
│   │   ├── models/            # 数据模型
│   │   ├── config/            # 配置加载
│   │   ├── cad/               # CAD 处理模块
│   │   ├── doc_gen/           # 文档生成模块
│   │   └── pipeline/          # 流水线模块
│   ├── tests/                 # 测试
│   └── pyproject.toml
├── documents/                  # 设计文档
├── documents_bin/              # 模板文件（docx/xlsx）
├── bin/                        # 第三方工具（ODA File Converter）
├── tools/                      # 辅助脚本
└── test/                       # 测试数据（已加入 .gitignore）
```

### 4.2 核心设计原则
1. **高度解耦**：模块通过 `interfaces.py` 定义的抽象接口通信
2. **单一职责**：每个文件只做一件事
3. **配置驱动**：所有业务规则和参数都从 YAML 配置读取，无硬编码
4. **失败隔离**：单图框失败不影响全局，只记录 flags/errors

### 4.3 模块职责划分

**模块 1：框架与配置层**
- `config/spec_loader.py`：加载业务规范
- `config/runtime_config.py`：运行期配置

**模块 2：DWG→DXF 与图框检测**
- `cad/oda_converter.py`：DWG↔DXF 转换
- `cad/frame_detector.py`：图框检测
- `cad/detection/`：检测算法子模块

**模块 3：图签字段提取**
- `cad/titleblock_extractor.py`：从图框提取字段

**模块 4：A4 多页成组**
- `cad/a4_multipage.py`：A4 说明图成组处理

**模块 5：裁切/拆分与输出**
- `cad/splitter.py`：图框裁切
- `cad/dxf_pdf_exporter.py`：PDF 导出
- `cad/autocad_pdf_exporter.py`：AutoCAD COM 导出

**模块 6：文档生成**
- `doc_gen/cover.py`：封面生成
- `doc_gen/catalog.py`：目录生成
- `doc_gen/design.py`：设计文件生成
- `doc_gen/ied.py`：IED 计划生成
- `doc_gen/pdf_engine.py`：PDF 导出引擎

**模块 7：打包与 API**
- `pipeline/job_manager.py`：任务管理
- `pipeline/executor.py`：流水线执行
- `pipeline/packager.py`：打包与 manifest
