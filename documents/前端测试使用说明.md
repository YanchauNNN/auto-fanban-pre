# 前端测试使用说明

## 1. 目标

本文档用于测试 `feature/frontend-blueprint` 分支当前版本的前端与真实后端联调结果。

当前前端已经切换为：

- 首页工作台
- 小型 `出图` 按钮
- `任务配置` 主弹窗
- `翻版配置` 次级弹窗
- 草稿保留与配置方案保存

当前真实可提交流程只有：

- `交付处理`

当前只做前端预留、不可真实提交的流程：

- `纠错`
- `翻版`

## 2. 工作目录

本说明基于下面这个 worktree：

```text
E:\project\auto-fanban-pre\.worktrees\frontend-blueprint
```

## 3. 启动前准备

建议开两个终端，工作目录都切到：

```powershell
E:\project\auto-fanban-pre\.worktrees\frontend-blueprint
```

注意：

- 只要后端代码有变更，测试前请先重启 API。
- 如果看到的页面行为、产物命名或下载状态明显还是旧版本，第一反应就是先重启 API。

## 4. 启动命令

### 4.1 启动后端 API

```powershell
uv run --project backend python -m uvicorn API.app.main:app --host 127.0.0.1 --port 8000
```

健康检查地址：

```text
http://127.0.0.1:8000/api/system/health
```

正常情况下应返回 JSON，并且至少看到：

- `ready: true`
- `storage_writable: true`
- `worker_alive: true`

### 4.2 启动前端

```powershell
cd E:\project\auto-fanban-pre\.worktrees\frontend-blueprint\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

前端地址：

```text
http://127.0.0.1:5173
```

默认 API 地址：

```text
http://127.0.0.1:8000
```

如果 API 不在默认地址：

```powershell
$env:VITE_API_BASE_URL="http://你的API地址"
npm run dev -- --host 127.0.0.1 --port 5173
```

## 5. 页面结构

首页现在分为三块：

- 左侧：系统状态、识别摘要、次级任务开关
- 中间：说明与主操作入口
- 右侧：最近任务

当前版本不再在首页直接铺开长表单。

所有参数录入都在 `任务配置` 弹窗内完成。

## 6. 基本操作流程

### 6.1 新建交付任务

1. 打开首页
2. 点击 `出图`
3. 在系统文件选择器中选择一个或多个 `.dwg`
4. 文件选择完成后，弹出 `任务配置`
5. 填写主参数
6. 如需查看更多非必填字段，点击 `展开高级选项`
7. 点击 `创建交付任务`
8. 成功后，最近任务列表会出现新任务

### 6.2 草稿恢复

1. 打开 `任务配置`
2. 填写部分参数后关闭弹窗
3. 返回首页后，页面会显示 `继续草稿`
4. 点击 `继续草稿`，应恢复上次未提交的文件和参数

注意：

- 关闭弹窗不会清空草稿
- 只有主动清空，或提交成功后，草稿才会重置

### 6.3 配置方案

当前版本支持配置方案的本地保存。

应测试：

- 保存方案
- 应用方案
- 方案重命名
- 删除方案

注意：

- 配置方案不会保存上传文件
- IED 编制日期到审定日期这组日期不会保存在方案里
- 这些日期进入弹窗时会默认填成当天

## 7. 任务配置弹窗说明

主弹窗主要分为：

- 文件摘要
- 识别摘要
- 主参数区
- 高级选项区
- 左侧次级任务开关

### 7.1 文件名项目号识别

前端会根据上传文件名做一层轻量识别：

- 规则：取文件名前四位数字作为项目号建议值
- 例如：`2016-xxx.dwg` 会识别为 `2016`

用途仅包括：

- 页面提示
- 项目号默认填充
- 翻版原始项目号建议值

它不会替代后端真实校验。

### 7.2 主参数与高级选项

规则如下：

- `required: true` 字段留在主参数区
- 命中 `required_when` 的字段也留在主参数区
- 只有 `required: false` 字段进入高级选项

### 7.3 次级任务开关

当前 UI 中保留了：

- `纠错`
- `翻版`

它们是互斥的，不允许同时生效。

但当前版本仍然只有 `交付处理` 可以真实提交。

如果点击 `纠错` 或 `翻版` 后继续提交，应看到“接口未开放”的阻断提示，而不是假成功。

## 8. 真实产物检查点

当前真实后端产物规则如下。

### 8.1 package.zip

`package.zip` 现在是平铺结构，不再分：

- `docs\`
- `drawings\`

压缩包根目录直接放最终交付文件。

注意：

- `manifest.json` 不进入压缩包
- `IED计划.xlsx` 单独下载，不进入压缩包

### 8.2 文档命名

当前封面与目录已经改成图纸式命名：

- 封面：`外部编码 + 版本 + 状态 + (内部编码)`
- 目录：`外部编码 + 版本 + 状态 + (内部编码)`

当前 `设计文件.xlsx` 仍保持原名：

- `设计文件.xlsx`

### 8.3 目录格式

目录 `xlsx` 当前检查点：

- `E` 列正文名称应为水平居中
- `E` 列正文名称应为垂直居中
- `wrapText = true`
- 行高采用分档逻辑，不再统一固定 `36`

当前分档规则：

- 1 到 2 行：`36`
- 3 行：`50`
- 4 行：`60`
- 5 行及以上：每多 1 行再加 `12`

注意：

- Excel 落盘后可能显示成 `50.1`、`60.0` 这类内部数值，这属于正常现象

## 9. 建议测试清单

### 9.1 首页与弹窗

- 首页不再出现三张主任务卡片
- 首页存在小型 `出图` 按钮
- 选择文件后弹出 `任务配置`
- 关闭弹窗后可通过 `继续草稿` 恢复

### 9.2 文件与识别

- 上传单个 `2016-xxx.dwg` 时，识别摘要显示 `2016`
- 多文件同项目号时，无冲突提示
- 多文件不同项目号时，出现冲突警示
- 重新选择文件后，识别结果同步刷新

### 9.3 表单与方案

- 必填字段显示 `必填`
- 高级选项默认收起
- 高级选项内只出现非必填字段
- 配置方案可保存、应用、重命名、删除
- 日期字段进入弹窗后默认是当天

### 9.4 次级任务

- `纠错` 与 `翻版` 互斥
- 点击 `翻版` 后弹出 `翻版配置`
- 翻版推荐项目号可回填
- 当前点击提交会提示接口未开放

### 9.5 真实提交

- `交付处理` 能真实创建任务
- 最近任务中能看到新任务
- 详情页中能看到状态、阶段、进度、flags、errors
- `package.zip` 和 `IED计划.xlsx` 下载按钮状态与后端返回一致

### 9.6 产物检查

建议至少核对以下内容：

- 压缩包是否为平铺结构
- `manifest.json` 是否未进入压缩包
- 封面与目录命名是否符合新规则
- 目录 `xlsx` 的 E 列是否居中
- 长标题行是否已升到 `50` / `60` 档

## 11. 常见排错

### 11.1 页面行为像旧版本

优先检查：

- 后端 API 是否已经重启
- 前端是否连到了正确的 API 地址

### 11.2 下载按钮是灰色

说明后端返回的对应产物当前不可用。

优先查看：

- `job.json`
- 任务详情里的 `flags`
- 任务详情里的 `errors`

### 11.3 压缩包结构不对

优先检查：

- 是否是旧任务目录
- 是否在后端改完后没有重启 API
