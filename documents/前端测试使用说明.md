# 前端测试使用说明

## 1. 目标

本文档用于主线仓库当前版本的前端联调与产物检查。

当前真实可提交流程有：

- `出图`
- `纠错`

当前仍未开放真实提交的流程有：

- `翻版`

## 2. 工作目录

当前主线仓库根目录：

```text
E:\project\auto-fanban-pre
```

## 3. 启动前准备

建议开两个终端，工作目录都切到：

```text
E:\project\auto-fanban-pre
```

注意：

- 只要后端代码有变更，测试前请先重启 API。
- 如果页面表现、下载状态或产物结构看起来还是旧逻辑，第一反应先重启 API。

## 4. 启动命令

### 4.1 启动后端 API

```powershell
cd E:\project\auto-fanban-pre
uv run --project backend python -m uvicorn API.app.main:app --host 127.0.0.1 --port 8000
```

健康检查地址：

```text
http://127.0.0.1:8000/api/system/health
```

正常情况下至少应看到：

- `ready: true`
- `storage_writable: true`
- `worker_alive: true`

### 4.2 启动前端

```powershell
cd E:\project\auto-fanban-pre\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

前端地址：

```text
http://127.0.0.1:5173
```

默认 API 地址：

```text
http://127.0.0.1:8000
```

如果 API 不在默认地址：

```powershell
cd E:\project\auto-fanban-pre\frontend
$env:VITE_API_BASE_URL="http://你的API地址"
npm run dev -- --host 127.0.0.1 --port 5173
```

## 5. 页面结构

首页当前分为三块：

- 左侧：系统状态、草稿入口、近期摘要
- 中间：说明与主操作按钮
- 右侧：最近任务

首页主按钮当前为：

- `出图`
- `纠错`

当前版本不再在首页直接铺开长表单，所有录入都在弹窗中完成。

## 6. 基本操作流程

### 6.1 出图

1. 打开首页
2. 点击 `出图`
3. 在系统文件选择器中选择一个或多个 `.dwg`
4. 文件选择完成后，打开 `任务配置` 弹窗
5. 填写主参数
6. 如需更多非必填字段，点击 `展开高级选项`
7. 点击 `创建交付任务`
8. 成功后，最近任务列表会出现新任务

### 6.2 纠错

1. 打开首页
2. 点击 `纠错`
3. 选择 `.dwg`
4. 在纠错弹窗中填写或选择 `project_no`
5. 点击提交
6. 成功后，最近任务列表会出现 `纠错` 类型任务

### 6.3 草稿恢复

当前支持两类草稿恢复：

- 出图草稿
- 纠错草稿

规则：

- 关闭弹窗不会清空草稿
- 再次打开同类弹窗时，会恢复未提交内容
- 主动清空或提交成功后，草稿才会重置

### 6.4 配置方案

当前仅 `出图` 弹窗支持配置方案：

- 保存方案
- 应用方案
- 重命名方案
- 删除方案

注意：

- 配置方案不保存上传文件
- IED 日期字段不保存到方案中
- 这些日期进入弹窗时默认就是当天

## 7. 出图弹窗说明

出图主弹窗当前包含：

- 文件摘要
- 识别摘要
- 主参数区
- 高级选项区
- 左侧摘要/快捷区

### 7.1 项目号识别

前端会根据文件名做一层轻量识别：

- 规则：取文件名前四位数字作为项目号建议值
- 作用：提示、默认回填、联动推荐

注意：

- 这只是前端建议值
- 最终仍以后端真实校验与处理为准

### 7.2 主参数与高级选项

规则如下：

- `required: true` 的字段留在主参数区
- 命中 `required_when` 的字段也留在主参数区
- 只有 `required: false` 的字段进入高级选项

### 7.3 组合输入控件

当前前端会按后端 `form-schema` 中的 `ui.widget` 通用处理组合输入，不再只为个别字段写死逻辑。

目前常见的可搜索组合框包括：

- `project_no`
- `file_category`
- `ied_design_type`
- `ied_responsible_unit`
- `ied_discipline_office`
- 以及后端后续继续下发为 `combobox` 的字段

## 8. 纠错弹窗说明

纠错弹窗只包含最小必要输入：

- `project_no`
- `dwg_files[]`

规则：

- `project_no` 使用可搜索下拉
- 可以手动清空，但会提示将依赖文件名推断
- 422 错误会直接显示在 `project_no` 下方
- 关闭弹窗后保留草稿

## 9. 下载与产物规则

### 9.1 出图任务

详情页应出现：

- `下载 package.zip`
- `下载 IED计划.xlsx`

规则：

- `package.zip` 是平铺结构
- 不再包含 `docs\`、`drawings\` 子目录
- 不包含 `manifest.json`
- `IED计划.xlsx` 单独下载，不进入 `package.zip`

### 9.2 纠错任务

详情页应出现：

- `下载 report.xlsx`

并展示摘要信息：

- `findings_count`
- `affected_drawings_count`
- `top_wrong_texts`
- `top_internal_codes`

## 10. 文档与命名检查点

### 10.1 封面与目录命名

当前封面与目录采用图纸式命名：

- `外部编码 + 版本 + 状态 + (内部编码)`

### 10.2 目录文件

当前目录 `xlsx` 需要重点检查：

- `D1:E1` 写入 `album_title_cn`
- `1818` 项目时，`D2:E2` 写入 `album_title_en`
- `D3:E3` 写入 `第{album_code}图册图纸(文件)目录`
- 第三行开始包含 `001`，不能直接从 `002` 开始
- `E` 列名称水平居中、垂直居中、自动换行
- 正文区从第 `9` 行开始按分档行高处理

当前行高分档规则：

- 1 到 2 行：`36`
- 3 行：`50`
- 4 行：`60`
- 5 行及以上：在 `60` 基础上每多 1 行加 `12`

说明：

- Excel 落盘后可能显示为 `50.1`、`60.0`，这属于正常现象

### 10.3 设计文件

当前设计文件 `xlsx` 需要重点检查：

- 包含 `001`
- `S` 列图幅走模板映射
- `N` 列专业走模板映射

## 11. 建议测试清单

### 11.1 首页

- 首页显示 `出图` 与 `纠错` 两个主按钮
- 不再显示旧的三张任务入口卡片

### 11.2 出图

- 选择 DWG 后弹出 `任务配置`
- 关闭弹窗后可通过草稿恢复继续填写
- 提交后任务列表出现新任务
- 详情页中 `package.zip` 和 `IED计划.xlsx` 下载状态正确

### 11.3 纠错

- 选择 DWG 后弹出纠错弹窗
- 可选择或清空 `project_no`
- 提交后任务列表出现 `纠错` 类型任务
- 详情页显示摘要信息和 `report.xlsx`

### 11.4 产物

至少检查：

- `package.zip` 是否为平铺结构
- `manifest.json` 是否未进入压缩包
- 封面与目录命名是否符合新规则
- 目录 `xlsx` 中 `album_title_cn / album_title_en / album_code` 是否落对位置
- 目录 `xlsx` 的 `E` 列是否居中且行高正确

## 13. 常见排错

### 13.1 页面表现像旧版本

优先检查：

- 后端 API 是否已经重启
- 前端是否连到了正确的 API 地址

### 13.2 下载按钮是灰色

说明对应产物当前不可用。

优先查看：

- `job.json`
- 任务详情里的 `flags`
- 任务详情里的 `errors`

### 13.3 压缩包结构不对

优先检查：

- 是否查看的是旧任务
- 是否在后端改完后没有重启 API

### 13.4 Word 弹出保存提示

这通常不是当前任务本身失败，而是本机残留的旧 `WINWORD.EXE` 进程导致的历史弹窗。

如果本轮任务已经完成，优先：

- 关闭残留 Word 窗口
- 确认没有手工打开未保存的 Word 文档
- 再重新执行烟测
