# 终端实装安装计划

## 目标

把 `fanban-terminal-deploy.rar` 解压到部署机后，按固定顺序完成以下工作：

1. 检查 IIS 代理前提。
2. 安装 IIS 代理组件。
3. 安装运行时依赖。
4. 初始化后端运行环境。
5. 配置 IIS 前端站点。
6. 注册后端常驻。
7. 运行健康检查。
8. 用固定样本做烟测。

本方案适用于以下场景：

- 只有部署机安装 AutoCAD 和 Office。
- 部署机不能联网。
- 其他客户端通过局域网浏览器访问部署机。

## 前提

部署机需要满足：

- 已安装 AutoCAD。
- 已安装 Office。
- 已安装 IIS。
- 可以从 U 盘拷贝 `fanban-terminal-deploy.rar`。

本计划假设你把压缩包解压到：

```text
D:\FanBanServer
```

如果你的实际路径不是这个，下面命令里的 `D:\FanBanServer` 要整体替换成真实路径。

## 步骤一：解压部署包

把压缩包：

```text
E:\project\auto-fanban-pre\build\fanban-terminal-deploy.rar
```

拷到部署机后，解压到：

```text
D:\FanBanServer
```

解压完成后，至少确认下面这些目录存在：

```text
D:\FanBanServer\frontend-dist
D:\FanBanServer\backend-runtime
D:\FanBanServer\install
D:\FanBanServer\scripts
D:\FanBanServer\documents\Resources
```

## 步骤二：先检查 IIS 代理前提

先不要急着继续部署，先确认 IIS 反代能力是否完整。

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
```

你会看到 JSON 输出，重点看这三个字段：

- `iis.installed`
- `url_rewrite.installed`
- `arr.installed`

### IIS 代理检查通过标准

如果结果是：

- `iis = true`
- `url_rewrite = true`
- `arr = true`

说明 IIS 反代前提已经满足，可以继续下一步。

如果 `url_rewrite` 或 `arr` 为 `false`，继续执行下一步离线安装。

## 步骤三：安装 IIS 代理组件

如果上一步显示 `URL Rewrite` 或 `ARR` 缺失，执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_iis_proxy_prereqs.ps1
```

这个脚本会先检查本机是否已安装：

- URL Rewrite。
- ARR。

只有缺失时才安装，不会无脑重复安装。

安装完后，再执行一次：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
```

确认三个字段都为 `true`。

## 步骤四：安装运行时依赖

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_runtime_prereqs.ps1
```

这个脚本现在会先检查本机状态：

- `.NET Framework 4.8` 或更高版本是否已安装。
- `VC++ 2015-2022 x64` 是否已安装。

只有缺失时才调用离线安装器。

### 运行时安装通过标准

你可能看到以下两类结果之一：

1. 本机已安装，脚本提示跳过。
2. 本机未安装，脚本开始安装。

只要不是“安装器不存在”的提示，就说明这一步逻辑正常。

## 步骤五：准备运行环境

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\prepare_terminal.ps1
```

这个脚本会完成：

1. 初始化 `storage`。
2. 探测 AutoCAD、Office、ODA、Resources。
3. 生成运行环境文件：

```text
D:\FanBanServer\scripts\runtime.env.ps1
```

### 运行环境准备通过标准

执行成功后，应至少看到：

- `runtime.env.ps1` 已生成。
- `logs\probe_target_env.json` 已生成。

如果失败，先把错误信息保存下来，再继续排查。

## 步骤六：配置 IIS 前端站点

推荐先用最简单的方式，直接通过部署机 IP 访问。

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\configure_iis_site.ps1 -SiteName FanBanTerminal -Port 80 -HostName ""
```

这个脚本会：

1. 创建或更新 IIS 站点。
2. 让 IIS 指向：

```text
D:\FanBanServer\frontend-dist
```

此外，如果 `URL Rewrite + ARR` 都已安装，则自动启用同源 `/api` 反代。

### 访问地址怎么定

如果你这样配置：

```powershell
-Port 80 -HostName ""
```

最终访问地址就是：

```text
http://部署机IP/
```

以后如果你想改成主机名，可以重新运行脚本，例如：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\configure_iis_site.ps1 -SiteName FanBanTerminal -Port 80 -HostName "fanban-server"
```

## 步骤七：注册后端常驻

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\register_backend_service.ps1
```

脚本会：

1. 优先尝试用 `nssm` 注册 Windows 服务。
2. 如果本机没有 `nssm`，回退成开机自启动计划任务。

这样后端能在部署机上常驻运行。

## 步骤八：运行健康检查

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\check_health.ps1
```

重点看三部分：

1. 本地环境探针。
2. IIS 代理前提。
3. `/api/system/health`。

如果最后能看到后端健康状态，说明前后端和后端服务都已经基本通了。

## 步骤九：部署机本机打开页面

在部署机浏览器里打开：

```text
http://127.0.0.1/
```

或者：

```text
http://部署机IP/
```

确认前端页面可以正常打开。

## 步骤十：另一台客户端验证访问

在同一局域网另一台电脑的浏览器里访问：

```text
http://部署机IP/
```

如果页面能打开，说明局域网访问已经通了。

## 步骤十一：做真实样本烟测

先用这两份固定样本验证系统：

1. `18185NE-JGS11.dwg`
2. `20261RS-JGS65.dwg`

目标是确认：

- 交付任务正常。
- 纠错任务正常。
- 联合任务正常。
- 结果可下载。

## 步骤十二：出问题时优先看哪里

如果部署过程中出问题，按这个顺序看：

1. `D:\FanBanServer\logs\probe_target_env.json`
2. `D:\FanBanServer\scripts\runtime.env.ps1`
3. `check_iis_proxy_prereqs.ps1` 的输出。
4. `check_health.ps1` 的输出。
5. 后端服务日志和任务日志。

## 完整命令顺序

完整命令顺序如下：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_iis_proxy_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_runtime_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\prepare_terminal.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\configure_iis_site.ps1 -SiteName FanBanTerminal -Port 80 -HostName ""
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\register_backend_service.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\check_health.ps1
```
