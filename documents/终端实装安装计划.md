# 终端实装安装计划

## 目标

把 `fanban-terminal-deploy` 解压到部署机后，按固定顺序完成以下工作：

1. 检查 IIS 代理前提。
2. 安装 IIS 代理组件。
3. 安装运行时依赖。
4. 初始化后端运行环境。
5. 执行深度环境检查。
6. 配置 IIS 前端站点。
7. 注册“登录时触发”的后端后台任务。
8. 运行健康检查。
9. 用固定样本做烟测。

本方案适用于以下场景：

- 只有部署机安装 AutoCAD 和 Office。
- 部署机不能联网。
- 其他客户端通过局域网浏览器访问部署机。
- 后端任务链依赖 Office COM，因此默认不再使用 `LocalSystem` Windows 服务常驻。

## 部署包类型

构建机会同时产出两类部署包：

1. 全量包：`build\fanban-terminal-deploy`
2. 局部包：`build\fanban-terminal-deploy-delta`

使用规则如下：

- 第一次部署，或者你不确定离线部署机当前基线时，必须使用全量包。
- 只有在离线部署机当前内容已经匹配“上一版全量包”时，才可以只拷贝局部包做迭代更新。
- 局部包根目录下只放需要覆盖的文件，`_delta\删除清单.txt` 里会列出需要手工删除的旧文件。
- 如果局部包 `_delta\使用说明.txt` 明确提示“未检测到上一版 full 包基线”，这次更新不要只拷局部包。

## 前提

部署机需要满足：

- 已安装 AutoCAD。
- 已安装 Office。
- 已安装 IIS。
- 可以从 U 盘拷贝部署包。
- 具备一个固定业务账号，用于登录后执行 Office COM 相关任务。

本计划假设你把部署包解压到：

```text
D:\FanBanServer
```

如果你的实际路径不是这个，下面命令里的 `D:\FanBanServer` 要整体替换成真实路径。

## 步骤一：解压部署包

第一次部署时，把全量包：

```text
E:\project\auto-fanban-pre\build\fanban-terminal-deploy
```

整体拷到部署机，放到：

```text
D:\FanBanServer
```

解压或覆盖完成后，至少确认下面这些目录存在：

```text
D:\FanBanServer\frontend-dist
D:\FanBanServer\backend-runtime
D:\FanBanServer\install
D:\FanBanServer\scripts
D:\FanBanServer\documents\Resources
```

如果你是在已经部署过的离线机上做版本迭代，并且确认当前机器就是上一版全量包状态，也可以只拷贝：

```text
E:\project\auto-fanban-pre\build\fanban-terminal-deploy-delta
```

这时做法是：

1. 先把局部包里的文件按相同相对路径覆盖到 `D:\FanBanServer`。
2. 再按照 `build\fanban-terminal-deploy-delta\_delta\删除清单.txt` 删除旧文件。
3. 覆盖完成后，再从“步骤四：安装运行时依赖”继续执行校验和健康检查。

## 步骤二：先检查 IIS 代理前提

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
```

重点看三个字段：

- `iis.installed`
- `url_rewrite.installed`
- `arr.installed`

### IIS 代理检查通过标准

如果结果是：

- `iis = true`
- `url_rewrite = true`
- `arr = true`

说明 IIS 反代前提已经满足，可以继续下一步。

如果 `url_rewrite` 或 `arr` 为 `false`，继续执行下一步离线安装。

## 步骤三：安装 IIS 代理组件

如果上一步显示 `URL Rewrite` 或 `ARR` 缺失，执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_iis_proxy_prereqs.ps1
```

安装完成后，再执行一次：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
```

确认三个字段都为 `true`。

## 步骤四：安装运行时依赖

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_runtime_prereqs.ps1
```

这个脚本现在只负责：

- 检查并安装 `.NET Framework 4.8`。
- 检查并安装 `VC++ 2015-2022 x64`。
- 解压部署包内的 `Python 3.13 embeddable runtime`。
- 同步随包 `site-packages` 到 `D:\FanBanServer\python-runtime\Lib\site-packages`。

这里已经不再包含任何 `NSSM` 相关安装步骤。

### 运行时安装通过标准

执行成功后，应至少看到类似输出：

- `.NET Framework 4.8 或更高版本已安装，跳过。`
- `VC++ 2015-2022 x64 运行时已安装，版本: ...，跳过。`
- `解压离线 Python 运行时: ...python-3.13.12-embed-amd64.zip`
- `已同步部署包 Python 依赖到: D:\FanBanServer\python-runtime\Lib\site-packages`

只要不是“安装器不存在”或“离线 Python 运行时准备失败”，就说明这一步逻辑正常。

## 步骤五：准备运行环境

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\prepare_terminal.ps1
```

这个脚本会完成：

1. 再次校验并补齐运行时依赖。
2. 初始化 `storage`。
3. 探测 AutoCAD、Office、ODA、Resources。
4. 生成运行环境文件：

```text
D:\FanBanServer\scripts\runtime.env.ps1
```

这个环境文件正常情况下不需要你手工再执行一遍。

原因是：

- 后端正式启动入口 `D:\FanBanServer\scripts\start_backend.ps1` 会自动加载它。
- 后面注册的“登录触发后台任务”最终调用的也是这个启动脚本。

只有在你想留在当前 PowerShell 会话里，手工复用这些环境变量时，才需要自己执行：

```powershell
. D:\FanBanServer\scripts\runtime.env.ps1
```

## 步骤六：执行深度环境检查

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\deep_check_terminal.ps1
```

输出文件在：

```text
D:\FanBanServer\logs\probe_target_env.deep.json
```

### 深度环境检查通过标准

至少应满足：

- `probe_target_env.deep.json` 已生成。
- `blocking issues = 0`。
- `Office status: pass (mode: deep)`。

这里要特别注意：

- 这一步验证的是“当前登录用户会话里的 Office 能力”。
- 它不等于 `LocalSystem` 服务态也能跑 Office COM。
- 正因为这个差异，正式部署默认改成“登录触发”的任务，而不是 Windows 服务。

## 步骤七：配置 IIS 前端站点

推荐先用最简单的方式，直接通过部署机 IP 访问。

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\configure_iis_site.ps1 -SiteName FanBanTerminal -Port 80
```

这个脚本会：

1. 创建或更新 IIS 站点。
2. 让 IIS 指向：

```text
D:\FanBanServer\frontend-dist
```

此外，如果 `URL Rewrite + ARR` 都已安装，则自动启用同源 `/api` 反代。

如果你准备直接按部署机 IP 访问，不要显式传 `-HostName ""`。

如果这里提示 `IIS 绑定冲突` 或 `0x800700B7`，通常说明 `80` 端口已经被另一个 IIS 站点占用，最常见就是 `Default Web Site`。

## 步骤八：注册“登录时触发”的后端后台任务

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\register_backend_task.ps1 -UserName "<本机登录账号>"
```

例如：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\register_backend_task.ps1 -UserName "wangrca"
```

这个脚本会：

1. 清理旧版同名 Windows 服务（如果存在）。
2. 注册一个“用户登录时触发”的计划任务。
3. 以交互用户令牌运行 `scripts\start_backend.ps1`。
4. 用隐藏窗口方式启动 PowerShell，不弹出常驻终端窗口。
5. 如果当前目标用户已经登录，会尝试立即启动任务。

### 这一步的关键要求

- `UserName` 必须是实际执行 Office 任务的固定业务账号。
- 该账号需要保持登录状态，Office COM 才能稳定工作。
- 这条链默认不再依赖 `NSSM`，也不再默认注册 Windows 服务。

如果你后面需要移除这条后台任务，执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\unregister_backend_task.ps1
```

## 步骤九：运行健康检查

执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\check_health.ps1
```

重点看三部分：

1. 本地环境探针。
2. IIS 代理前提。
3. `/api/system/health`。

如果最后能看到后端健康状态，说明前后端和后台任务都已经基本通了。

## 步骤十：部署机本机打开页面

在部署机浏览器里打开：

```text
http://127.0.0.1/
```

或者：

```text
http://部署机IP/
```

如果你把 IIS 绑定到了 `8080`，就改成：

```text
http://127.0.0.1:8080/
```

## 步骤十一：另一台客户端验证访问

在同一局域网另一台电脑的浏览器里访问：

```text
http://部署机IP/
```

如果页面能打开，说明局域网访问已经通了。

如果你想改成主机名访问，例如：

```text
http://fanban-server/
```

则必须额外满足下面之一：

1. 局域网 DNS 已经把 `fanban-server` 解析到部署机 IP。
2. 部署机和客户端都手工配置了 `hosts`。

## 步骤十二：做真实样本烟测

先用这两份固定样本验证系统：

1. `18185NE-JGS11.dwg`
2. `20261RS-JGS65.dwg`

目标是确认：

- 交付任务正常。
- 纠错任务正常。
- 联合任务正常。
- 结果可下载。
- 封面、目录、设计文件、IED 产物符合预期。

## 步骤十三：出问题时优先看哪里

如果部署过程中出问题，按这个顺序看：

1. `D:\FanBanServer\logs\probe_target_env.json`
2. `D:\FanBanServer\logs\probe_target_env.deep.json`
3. `D:\FanBanServer\scripts\runtime.env.ps1`
4. `check_iis_proxy_prereqs.ps1` 的输出。
5. `check_health.ps1` 的输出。
6. 任务计划程序里 `FanBanBackend` 的执行记录。
7. 后端任务日志和 `storage\jobs` 下的作业产物。

## 完整命令顺序

完整命令顺序如下：

```powershell
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\check_iis_proxy_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_iis_proxy_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\install_runtime_prereqs.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\prepare_terminal.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\deep_check_terminal.ps1
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\configure_iis_site.ps1 -SiteName FanBanTerminal -Port 80
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\install\register_backend_task.ps1 -UserName "<本机登录账号>"
powershell -ExecutionPolicy Bypass -File D:\FanBanServer\scripts\check_health.ps1
```
