## 1. 一句话结论

- 模块5已完成“可用版”实现：裁切、命名、DWG 导出、PDF 导出、A4 多页、失败隔离、单测回归均已跑通。
- 当前纯 Python PDF 栈（`ezdxf + matplotlib`）在**字体/字高/插入点精确复刻**方面存在理论上限，无法稳定达到 CAD 原生一致性。
- 下一步应做：**增量引入 AutoCAD 原生打印链路（兼容 2021/2022）**，与现有实现并存，支持 A/B 对比。

## 2. 原始目标与最终边界

### 2.1 原始业务目标（模块5）

- 把单个 DXF 按图框切分为多个中间 DXF。
- 输出 DWG 与 PDF，文件名规则：`external_code(internal_code)`。
- A4 多页场景输出多页 PDF，保留页序；必要时可兜底单页。
- 裁切后坐标保持原坐标系（不归零、不平移）。

### 2.2 关键质量目标

- 图框内图素零误删（硬约束）。
- 图框外图素不应进入最终 PDF 可视范围（严格窗口打印）。
- 页边距遵循 `pdf_margin_mm`。
- 按标准图幅 1:1 输出（不跟随放大后图素尺寸）。

### 2.3 仍未完全满足的项（当前纯 Python 栈）

- 字体（尤其 SHX/bigfont）显示与 CAD 原版一致性。
- 部分文本字高/位置存在偏移。

## 3. 版本迭代过程（必须了解）

### V1：新建 DXF + `add_foreign_entity`

- 思路：新建目标文档，仅导入框内实体。
- 结果：实体丢失（尤其块参照相关）、样式复刻不稳定。
- 结论：废弃。

### V2：新建 DXF + `ezdxf.addons.Importer`

- 思路：补齐块定义导入，缓解 V1 丢失问题。
- 结果：实体完整性改善，但字体仍明显偏离原图。
- 结论：仍不满足“完全复刻”。

### V3（当前主干裁切策略）：复制原 DXF + 删除框外实体

- 思路：`shutil.copy2` 保留原文件全部结构，再删除框外实体。
- 优势：字体/样式/块定义/头段最大限度保留，视觉最接近原图。
- 当前在 `backend/src/cad/splitter.py` 落地并成为主策略。

### V4：Stage B 流水线拆分（clip/export 解耦）

- `SPLIT_AND_RENAME`：仅裁切产出中间 DXF。  
- `EXPORT_PDF_AND_DWG`：统一导出 DWG/PDF。  
- 位置：`backend/src/pipeline/executor.py`。

### V5：PDF 严格窗口 + 1:1 图幅 + 线宽映射

- 单页/多页都走窗口打印。
- 页面尺寸按标准图幅（`paper_variants.W/H`）+ `pdf_margin_mm`。
- 黑白 + ACI 线宽：`1 -> 0.4mm`，`2~255 -> 0.18mm`。
- 位置：`backend/src/cad/dxf_pdf_exporter.py`、`splitter.py`、`executor.py`。

### V6：字体增强尝试（仍非最终解）

- 接入 `fronts/Fonts` 到 `ezdxf.options.support_dirs`。
- 注册字体目录下 TTF/TTC 到 matplotlib。
- 增加“渲染前样式归一化”将高风险样式映射到 `SimSun`。
- 结果：方块字有所缓解，但位置/度量仍难与 CAD 原生一致。
- 结论：纯 Python 栈到达上限，应上 AutoCAD 原生打印。

## 8. 已知问题与根因（必须给新AI）

### 问题 A：纯 Python PDF 与 CAD 原图字体/位置不一致

- 根因：
  - CAD 文本使用 `SHX/bigfont` 度量体系；
  - 当前渲染依赖 matplotlib/TTF 度量；
  - 即使补齐字体和样式映射，仍会出现字高、字距、插入点偏移。

### 问题 B：个别脚本场景会出现同名覆盖/偶发路径错误

- 主要发生在测试脚本级（非主流程核心逻辑）。
- 核心 pipeline 中已具备失败隔离与 flag 机制。

### 问题 C：1818 样例 `external_code` 个别提取依赖上游模块3质量

- 该问题在历史中出现过（INSERT 字符分散提取），属于模块3/图签提取侧风险项。
