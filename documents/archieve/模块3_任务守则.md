## 2) 必须满足的功能点（Acceptance）

### 2.1 ROI 还原（必须按规范公式）

- 坐标系：`rb_offset_scaled`
- 还原公式：
  - `xmin = outer_xmax - dx_left * sx`
  - `xmax = outer_xmax - dx_right * sx`
  - `ymin = outer_ymin + dy_bottom * sy`
  - `ymax = outer_ymin + dy_top * sy`

### 2.2 字段提取与解析（必须覆盖）

| 字段 | 规则 |
|---|---|
| `external_code` | 固定19位，去除 DOC.NO，A-Z0-9 |
| `internal_code` | 支持 `XXXXXXX-XXXXX-XXX` 与短格式 |
| `engineering_no` | 4位数字 |
| `subitem_no` | 文本或词库 |
| `paper_size_text` | 直接文本 |
| `discipline` | 直接文本 |
| `scale_text` | 正则 `^1[:：](\\d+(?:\\.\\d+)?)$` |
| `page_total/page_index` | 解析 `共N张 第M张`；M 支持 X/x |
| `title_cn/title_en` | 按 y 聚类成行，CJK 为中文，英文数字为英文 |
| `revision/status/date` | 列内 **y 最大** 的文本 |

### 2.3 比例一致性检查（不中断）

- 比较 `geom_scale_factor` 与 `scale_text` 解析出的分母
- 若不一致：追加 flag `比例不一致`（按规范）

### 2.4 失败策略

- **不抛异常中断任务**
- 缺失字段置空，告警写入 flags

---

## 3) 实现步骤（建议）

1. **加载规范**
   - `spec = load_spec()`
   - 取 `roi_profiles` 和 `field_definitions`

2. **ROI还原**
   - 用 `frame.runtime.roi_profile_id` 选择 BASE10/SMALL5
   - 对每个 ROI 计算绝对坐标

3. **提取文本**
   - 遍历 `TEXT` / `MTEXT`
   - 获取插入点坐标
   - 过滤进入 ROI
   - 保存：`text/x/y/height/align`

4. **字段解析**
   - 按 `field_definitions` 的 parse 规则解析
   - 写入 `FrameMeta.titleblock`

5. **一致性检查**
   - 比例不一致 → `flags`
