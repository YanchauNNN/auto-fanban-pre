# 交付后端任务

## 背景

当前主线已经支持：

- 单次提交创建一个 `TaskGroup`
- 组内自动创建：
  - `deliverable_main`
  - `audit_check`
- 前端任务列表已经按“任务包卡片”展示

但点进任务包详情后，子任务区域仍然偏“执行诊断页”，不符合当前产品目标。  
当前后端虽然已经能提供：

- 交付产物目录与 `manifest.json`
- 纠错 `report.json / report.xlsx`

但 API detail 还没有把“前端真正需要展示的结果摘要”整理出来。

同时，当前 flags 里仍有一些应该被视为允许路径或已自动修复的内容在前端继续暴露：

- `PLOT_FROM_SOURCE_WINDOW`
- `PLOT_WINDOW_USED`
- `PAPER_SIZE_MISMATCH`

其中：

- `PLOT_FROM_SOURCE_WINDOW`、`PLOT_WINDOW_USED` 应视为允许路径，不再上报给用户
- `PAPER_SIZE_MISMATCH` 如果已经出现对应的 `PAPER_SIZE_AUTO_FIXED`，则只保留 `PAPER_SIZE_AUTO_FIXED`

## 本轮目标

让前端在任务包详情中，能够只展示“结果”，不再展示“执行过程诊断”。

具体要达成两件事：

1. **交付子任务**能直接展示：
   - 拆出了多少个 `dwg`
   - 拆出了多少个 `pdf`
   - 每张图的文件名
   - 每张图页数
2. **纠错子任务**能直接展示：
   - 有哪些错误文本
   - 每个错误分别出现在哪些 `internal_code` 图纸中

## 必做后端改动

### 1. 交付子任务 detail 返回结果摘要

当前来源已经存在于：

- `manifest.json`
- `context.frames`
- `context.sheet_sets`

需要在交付任务 detail 序列化中新增结构化字段，供前端直接消费。

建议字段：

```json
{
  "deliverable_outputs": {
    "dwg_count": 11,
    "pdf_count": 11,
    "documents": [
      {"name": "封面.docx", "kind": "docx"},
      {"name": "封面.pdf", "kind": "pdf"},
      {"name": "目录.xlsx", "kind": "xlsx"},
      {"name": "目录.pdf", "kind": "pdf"},
      {"name": "设计文件.xlsx", "kind": "xlsx"}
    ],
    "drawings": [
      {
        "name": "XZ1RSL32002B25C42SDACFC (20261RS-JGS65-002)",
        "internal_code": "20261RS-JGS65-002",
        "dwg_name": "XZ1RSL32002B25C42SDACFC (20261RS-JGS65-002).dwg",
        "pdf_name": "XZ1RSL32002B25C42SDACFC (20261RS-JGS65-002).pdf",
        "page_total": 1
      },
      {
        "name": "XZ1RSL32001B25C42SDACFC (20261RS-JGS65-001)",
        "internal_code": "20261RS-JGS65-001",
        "dwg_name": null,
        "pdf_name": "XZ1RSL32001B25C42SDACFC (20261RS-JGS65-001).pdf",
        "page_total": 4
      }
    ]
  }
}
```

说明：

- `single_frame` 的 `page_total` 固定为 `1`
- `a4_sheet_set` 的 `page_total` 使用 `sheet_set.page_total`
- 这部分信息不要让前端自己扫目录或读 manifest 文件
- 直接在 `GET /api/jobs/{job_id}` detail 中返回

推荐修改位置：

- `API/app/runtime.py`
- `backend/src/pipeline/packager.py`

### 2. 纠错子任务 detail 返回“错误 -> 图纸编号”映射

当前 `report.json` 的 `findings` 已经有：

- `matched_text`
- `internal_code`

所以后端已经具备“一一对应”的原始数据，只需要在 detail 返回里补一个聚合字段。

建议新增字段：

```json
{
  "finding_groups": [
    {
      "matched_text": "2016",
      "count": 1,
      "internal_codes": ["20261RS-JGS65-001"]
    },
    {
      "matched_text": "漳州",
      "count": 2,
      "internal_codes": ["20261RS-JGS65-001", "20261RS-JGS65-003"]
    }
  ]
}
```

要求：

- 按 `matched_text` 聚合
- `internal_codes` 去重后返回
- 保留 `count`
- 顺序按 `count` 倒序，前端可直接展示

推荐修改位置：

- `backend/src/audit_check/reporting.py`
- `backend/src/audit_check/executor.py`
- `API/app/runtime.py`

### 3. flags 降噪规则

当前前端看到的 flags 里，以下两类需要在后端清理：

#### 3.1 直接过滤

以下 flag 不再上报给用户：

- `PLOT_FROM_SOURCE_WINDOW`
- `PLOT_WINDOW_USED`

它们属于允许路径，不再作为用户告警展示。

#### 3.2 成对降噪

如果同一个 drawing / sheet_set 下同时存在：

- `PAPER_SIZE_MISMATCH`
- `PAPER_SIZE_AUTO_FIXED`

则最终只保留：

- `PAPER_SIZE_AUTO_FIXED`

也就是说：

- 自动修复成功后，不再继续显示 `PAPER_SIZE_MISMATCH`

#### 3.3 生效范围

降噪要统一作用于以下输出：

- job detail 的 `flags`
- group detail 的 `flags`
- `manifest.json` 的 drawing flags

推荐修改位置：

- `backend/src/cad/cad_dxf_executor.py`
- `backend/src/pipeline/executor.py`
- 必要时 `API/app/runtime.py` 增加最终序列化过滤

## API 期望变化

### 单个交付任务 detail

新增：

- `deliverable_outputs`

### 单个纠错任务 detail

新增：

- `finding_groups`

### 任务包 detail

子任务 `children` 中，对应 child detail 需要能够在单独详情页里拿到上述字段。  
如果当前 group detail 只返回 summary 级 children，不要求直接在 group detail 里内联全部结果结构；前端可以继续点击子任务详情页。

## 前端将据此做的展示

后端完成后，前端会按以下方式展示：

### 交付子任务

- 不再显示：
  - 当前进度
  - 打印样式
  - slot
  - 路径
  - ctb / pmp / pc3
- 只显示：
  - `dwg_count`
  - `pdf_count`
  - 图纸清单
  - 文档清单
  - 每张图页数

### 纠错子任务

- 不再显示执行诊断
- 只显示：
  - 总错误数
  - 受影响图纸数
  - `matched_text -> internal_codes`
