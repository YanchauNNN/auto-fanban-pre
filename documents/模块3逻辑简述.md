## 1) 已实现功能

### 1.1 核心类

- **`TitleblockExtractor`**（`backend/src/cad/titleblock_extractor.py`）
  - 实现接口：`ITitleblockExtractor`
  - 唯一公开方法：`extract_fields(dxf_path: Path, frame: FrameMeta) -> FrameMeta`
  - 构造参数：`spec_path: str | None = None`（默认从 `documents/参数规范.yaml` 加载）

### 1.2 功能清单

| 功能 | 说明 |
|---|---|
| ROI 还原 | 按 `rb_offset_scaled` 公式从 `outer_bbox + sx/sy` 还原绝对 ROI 坐标 |
| 锚点文本验证 | 提取前检查图框 ROI 内是否命中锚点文本，未命中则追加 flag `未命中锚点文本` 并跳过 |
| 字段解析 | `external_code`(19位) / `internal_code`(正则) / `engineering_no` / `subitem_no` / `paper_size_text` / `discipline` / `scale_text` / `page_info` / `title_cn+title_en`(中英文分流) / `revision` / `status` / `date` |
| A4 从属帧页码提取 | A4 图框（无锚点文本的从属帧）只提取右上角页码标记，不做完整 titleblock 解析 |
| 比例一致性检查 | `geom_scale_factor` vs `scale_denominator`，不一致时追加 flag `比例不一致` |
| 容错策略 | 任何字段缺失不抛异常，置空并记录到 `raw_extracts` |

### 1.3 A4 逻辑分流

```text
extract_fields(frame)
  ├─ A4 图框?
  │   ├─ 有锚点文本 → 完整 titleblock 解析（A4 主帧/Master）
  │   └─ 无锚点文本 → 仅提取右上角页码（A4 从属帧/Slave）
  │         └─ 先查 ROI 内 `共N张 第M张` / `X OF Y` / `rolls X OF Y`
  │         └─ 若空则 fallback（限定图框边界内文本）
  └─ 非 A4 图框
      ├─ 有锚点文本 → 完整 titleblock 解析
      └─ 无锚点文本 → 追加 flag `未命中锚点文本`，跳过
```

### 1.4 本次修复的 Bug

- **fallback 跨帧误取页码**：`_fallback_a4_page_marker` 原先遍历整个 DXF 的所有文本，未限制在当前图框边界内，导致其他帧的页码文本被错误采纳。修复后增加了图框边界检查（含 5% 容差）。

## 2) 数据流（模块2 → 模块3 → 模块4）

```text
模块2 输出                    模块3 填充                   模块4 消费
──────────────────────        ──────────────────────       ──────────────────────
FrameMeta.runtime             FrameMeta.titleblock         frames + titleblock
  .frame_id                     .internal_code             → group_a4_pages()
  .source_file                  .external_code             → (non_a4_frames,
  .outer_bbox                   .engineering_no                sheet_sets)
  .paper_variant_id             .subitem_no
  .sx / .sy                     .paper_size_text
  .geom_scale_factor            .discipline
  .roi_profile_id               .scale_text / .scale_denominator
  .flags[]                      .page_total / .page_index
                                .title_cn / .title_en
                              FrameMeta.raw_extracts       .revision / .status / .date
                              FrameMeta.runtime.flags      .flags（追加，不覆盖）
                                (追加 比例不一致 等)
```

### 模块4 可直接使用的关键字段

- `frame.runtime.paper_variant_id`：判断是否为 A4 图框（`"A4" in paper_variant_id.upper()`）
- `frame.runtime.outer_bbox`：用于计算 A4 帧间距离/邻接关系
- `frame.titleblock.page_total` / `page_index`：A4 主帧和部分从属帧已填充
- `frame.titleblock.internal_code`：A4 主帧有值，从属帧为 None
- `frame.runtime.flags`：已包含 `未命中锚点文本`、`比例不一致` 等

## 4) pytest 编写规范（本项目约定）

### 4.1 目录结构

```text
backend/tests/
  conftest.py          ← 公共 fixtures 和工具函数
  __init__.py
  unit/
    __init__.py
    test_<module>.py   ← 每个被测模块对应一个文件
```

### 4.2 conftest.py 可用资源

| Fixture / Helper | Scope | 用途 |
|---|---|---|
| `spec` | session | 加载真实 `参数规范.yaml`（或 mock） |
| `anchor_spec` | session | 锚点定位器测试用 BusinessSpec（含 ROI/anchor 完整配置） |
| `runtime_config` | function | 运行期配置 |
| `sample_bbox` | function | 示例 BBox(0,0,841,594) |
| `sample_titleblock` | function | 完整填充的示例 TitleblockFields |
| `sample_frame` | function | 示例 FrameMeta（含 runtime + titleblock） |
| `sample_global_params` | function | 示例 GlobalDocParams |
| `sample_doc_context` | function | 示例 DocContext |
| `temp_job` | function | 临时 Job（自动清理） |
| `temp_dir` | function | 临时目录 |
| `sample_dxf_path` | function | 最小 DXF 文件路径 |
| `dxf_doc` | function | 返回 `(doc, msp)` 元组 |
| `add_rect_polyline()` | 纯函数 | 向 msp 添加闭合矩形多段线 |
| `add_rect_lines()` | 纯函数 | 向 msp 添加 4 条 LINE 围成的矩形 |

### 4.3 编写原则

1. **一个测试函数测一个行为**，测试名 `test_<被测行为>` 或 `test_<场景>_<预期>`
2. **数据变体用 `@pytest.mark.parametrize`**，而非复制粘贴测试函数
3. **共享的 setup（spec/ezdxf doc）用 conftest fixture**，文件专用的 helper 保留在文件内
4. **class 内多个测试共享同一 fixture 时**，用 `@pytest.fixture` 在 class 内定义（如 `derived_2016`）
5. **Dummy/Mock 类仅在单文件使用时不上提 conftest**（如 `DummyFinder`/`DummyFitter`）
6. **不减少覆盖度**：参数化只是把同构的 N 个函数变成 1 个函数 + N 组参数
7. **每次修改源码后运行**：`python -m pytest backend/tests/unit/ -v --tb=short`
8. **从项目根目录运行**（不是 `backend/`），确保 `documents/参数规范.yaml` 路径正确

### 4.4 Ruff 要求

- 每次提交前执行 `python -m ruff check <files>`
- import 排序使用 `ruff check --fix` 自动修复
- 不允许 `W293`（空行含空格）、`I001`（import 未排序）
