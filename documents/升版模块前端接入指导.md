# 升版模块前端接入指导

## 目标
- 用新的升版模型替换旧的四个输入项：`upgrade_start_seq`、`upgrade_end_seq`、`upgrade_revision`、`upgrade_note_text`。
- 新模型只保留两个字段：
  - `is_upgrade`
  - `upgrade_sheet_codes`
- 升版只控制目录文件 `I` 列标记，不参与任何版本号计算。

## 交互要求
- 在高级选项中新增主开关：`是否升版`。
- 默认关闭。
- 打开后，在开关旁边展开次级配置框。
- 次级配置只保留一个输入项：`升版图纸编号`。
- 关闭开关时：
  - 次级配置框隐藏。
  - 已输入的 `upgrade_sheet_codes` 不能丢失，重新打开时要恢复。

## 字段定义
### `is_upgrade`
- 类型：字符串，提交值为 `"true"` 或 `"false"`
- 默认：`"false"`
- 含义：是否启用升版标记

### `upgrade_sheet_codes`
- 类型：字符串
- 默认：空字符串
- 含义：匹配图纸 `internal_code` 最后三位的升版编号集合
- 允许为空：为空时表示“仅目录文件本身标记为升版，不标记任何图纸行”

## 输入框提示文案
悬浮提示固定为：

```text
输入图纸内部编码最后三位，支持单个编号和区间组合。
示例：001~099、001、003、005~009
支持分隔符：、 ， , 。 . ； ;
支持连接符：~ 和 -
留空表示仅标记目录文件本身为升版。
```

## 输入规则
- 支持单值：`001`
- 支持区间：`001~099`
- 支持组合：`001、003、005~009`
- 支持 1 到 3 位数字：`1,3,5-9`
- 后端会自动左补零到 3 位
- 非法片段需要阻止提交，并展示后端返回的字段错误

## 提交格式
继续走现有 `draft.values` 提交结构，新增/保留：

```json
{
  "is_upgrade": "true",
  "upgrade_sheet_codes": "001、003、005~009"
}
```

关闭升版时：

```json
{
  "is_upgrade": "false",
  "upgrade_sheet_codes": ""
}
```

## 后端行为说明
- `is_upgrade = false`
  - 封面行、目录文件 `I` 列和图纸行全部留空
- `is_upgrade = true` 且 `upgrade_sheet_codes = ""`
  - 仅目录文件那一行写标记
  - 图纸行不写标记
- `is_upgrade = true` 且有编号
  - 封面行与封面行与目录文件那一行固定写标记
  - 命中编号的图纸行写同样标记
- 标记文本：
  - 非 `1818`：`升版`
  - `1818`：`升版 upgrade`

## 与旧模型的区别
以下旧控件需要从 UI 中移除，不再展示、不再提交：
- 升版起始号
- 升版结束号
- 升版版本
- 升版标记

## 配置保存要求
- `is_upgrade`
- `upgrade_sheet_codes`

这两个字段都必须进入当前任务配置草稿、预设和本地保存结构。

## 验收点
- 默认不升版
- 勾选后出现次级配置框
- 关闭后再打开，之前输入仍保留
- 提交时不再发送旧四字段
- 后端返回字段错误时，错误定位到 `upgrade_sheet_codes`
- `1818` 项目展示/提交后，目录标记语义按 `升版 upgrade` 理解
