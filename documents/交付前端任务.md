# 交付前端任务

## 目标

后端已经支持“单次提交生成一个 `TaskGroup`，并在组内自动创建交付子任务 + 纠错子任务”。
前端需要从“先 `createBatch`，再 `createAuditCheck`”的旧模式，切换到“一次 `createBatch(run_audit_check=true)`”的新模式。

同时，后端已经接入打印样式 key 模型，后续前端需要支持：

- 系统默认打印样式
- 单任务覆盖打印样式

## 本轮必须完成的前端改动

### 1. 联合提交改成一次 `createBatch`

当前旧逻辑：

1. 先调用 `createBatch(...)`
1. 如果勾选纠错，再调用 `createAuditCheck(...)`

新逻辑：

1. 如果用户未勾选纠错：
   - 调用 `createBatch(params, files, false)`
1. 如果用户勾选纠错：
   - 直接调用 `createBatch(params, files, true)`
   - 不再单独调用 `createAuditCheck(...)`

说明：

- `createAuditCheck(...)` 仍保留，仅用于“纯纠错工作台”
- 交付工作台不再走“二次提交”模式

### 2. API 类型扩展

需要更新前端 API 类型定义，让 `createBatch` 支持第三个参数：

```ts
createBatch(
  params: Record<string, string>,
  files: File[],
  runAuditCheck?: boolean,
): Promise<CreateBatchPayload>
```

提交到后端时，需要在表单里追加：

- `run_audit_check=true`

### 3. 任务列表切换到 `TaskGroup` 视图

后端 `GET /api/jobs` 现在可能返回两类条目：

- 普通 job
- `TaskGroup`

新增/兼容字段：

- `is_group`
- `group_id`
- `source_filenames`
- `run_audit_check`
- `child_job_ids`
- `children`（detail 接口）
- `task_role`
- `shared_run_id`
- `plot_style_key`
- `plot_resource_mode`
- `slot_id`
- `cad_version`
- `accoreconsole_exe`
- `profile_arg`
- `pc3_path`
- `pmp_path`
- `ctb_path`

前端要求：

1. `is_group=true` 的项显示成“顶层任务卡片”
1. 卡片里显示两个子任务 chip：
   - `交付`
   - `纠错`
1. 子任务状态从 `children` 或详情接口读取

### 4. 详情页改成 group + children 双层视图

对于 `TaskGroup`：

- 顶层显示 group 阶段：
  - `PREP_SOURCE`
  - `DELIVERABLE_BRANCH`
  - `AUDIT_BRANCH`
  - `DOCS_AND_PACKAGE`
  - `GROUP_COMPLETE`
- 下方显示子任务摘要：
  - `deliverable_main`
  - `audit_check`

对于子任务：

- 继续显示原有单任务详情
- 显示本次执行诊断：
  - `plot_style_key`
  - `plot_resource_mode`
  - `slot_id`
  - `cad_version`
  - `pc3_path`
  - `pmp_path`
  - `ctb_path`

### 5. 打印样式选择 UI

后端已经支持稳定 key，不直接依赖前端传文件名。
前端展示中文名，提交稳定 key。

#### 选项映射

- `红色更宽` -> `red_wider`
- `同线宽` -> `same_width`
- `交审图` -> `review_white`

#### 行为要求

- 默认值：读取系统默认打印样式
- 允许用户在单任务中覆盖
- 提交给后端的是：`plot_style_key`
- 不直接提交 `ctb` 文件名

建议展示位置：

- 任务创建表单高级选项
- 或任务配置弹窗中的“打印设置”区块

### 6. 下载入口调整

`TaskGroup` 详情页需要区分：

- 交付产物
- 纠错报告

建议 UI：

- 下载交付包
- 下载 IED
- 下载纠错报告

## 联调验收

### 场景 1：仅交付

- 提交 1 个 DWG
- `runAuditCheck = false`
- 预期：仍生成普通交付任务

### 场景 2：交付 + 纠错

- 提交 1 个 DWG
- `runAuditCheck = true`
- 预期：
  1. 前端只发起一次 `/api/jobs/batch`
  1. 不再单独发 `/api/jobs/audit-replace`
  1. 列表中出现 1 个 `TaskGroup` 卡片
  1. 卡片内有两个子任务 chip：交付 / 纠错

### 场景 3：打印样式覆盖

- 选择 `同线宽`
- 提交后查看任务详情
- 预期：
  - `plot_style_key = same_width`
  - `ctb_path` 指向 `fanban_monochrome-same width.ctb`

### 场景 4：纯纠错工作台

- 继续单独调用 `/api/jobs/audit-replace`
- 预期：不受这次改动影响

## 注意事项

1. 前端不要自己拼接 group 结构，以后端返回为准。
1. `run_audit_check` 只在交付工作台的联合提交里使用。
1. 纯纠错工作台不要改成 group 模式。
1. 打印样式 UI 传的是稳定 key，不是文件名。
